テスト技法

[値の決め方]
同値クラス	範囲内でどの値を選んでも結果は同じ
境界値テスト	同値クラスの境界、すぐ下、すぐ上の値

テストケースの作成	複数の入力項目を同時にテストする
	・全て有効値
	・１つの無効値とそれ以外は有効値

	・境界値の上限と下限の全ての組み合わせ
	・境界値のすぐ下か上の無効値とそれ以外は有効値
	・有効値は境界値とする

[テストケースの作り方]
デシジョンテーブル	ルール：条件１〜ｎの時、アクション１〜ｎが起こる
				それ以外の有効値はDC(Don't Care)とする
				ルール１	...	ルールｎ
	条件
		条件１
		...
		条件ｎ

	アクション
		アクション1
		...
		アクションｎ

	ルール		テストケース
	条件		入力
	アクション	結果

状態遷移表	イベントをシステムの応答と一緒に文書化する
		全ての遷移を網羅するテストケースが望ましい

	状態	イベント	アクション	次の状態
	初期状態	イベント１	アクション１	状態１
	初期状態	...	...	...

	状態１	イベント１１	アクション１１	状態２
	状態１	...	...	...

	状態ｎ	...	...	...

	状態	システムがイベントを待っている状況
	遷移	イベントによって別の状態へ変わる
	イベント	システムの状態を変更させる何か
	アクション	状態の変更によって引き起こされる操作

[ペアテスト]
直交表	２次元の表で、任意の２つの列に値の全てのペアの組み合わせがある
		列１	...	列ｎ
	行１
	...
	行ｎ

	作り方	１　各変数の値を調べる
		２　列数は変数の数と同じ以上
		３　行数は各変数の値の数
			ex)	8*1, 6*1, 3*1 →  8*2, 4*1	※ちょうどいい大きさに合わせる
				行数 = 8 * 8 = 64
				全組み合わせ = 8 * 6 * 3 = 144
		４　条件を満たす直交表を検索する（書籍、Web、ツール）
		５　直交表にテストを当てはめる
		６　余剰のセルは適当な有効値を割り当てる

全ペアアルゴリズム	全ペアを生成するツール
			直交表と同じような効果
			直交表も全ペアアルゴリズムも特定の非常に重要な組み合わせがあれば、独自に追加する必要がある
